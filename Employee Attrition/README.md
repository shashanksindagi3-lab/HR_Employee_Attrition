# HR Attrition & Income Predictor

Two scikit-learn models trained on the IBM HR Employee Attrition dataset, served through a Streamlit app:

| Model | Type | Target | Test performance |
|---|---|---|---|
| Decision Tree Classifier | Classification | `Attrition` (Yes/No) | ~73% accuracy |
| Linear Regression | Regression | `MonthlyIncome` | RВІ в‰€ 0.94, MAE в‰€ $890 |

## Run locally

```bash
git clone <your-repo-url>
cd <your-repo>
pip install -r requirements.txt

# (Re)train the models вЂ” writes models/*.pkl and models/*_meta.json
python train.py

# Launch the app
streamlit run app.py
```

The app opens at `http://localhost:8501`. Adjust the employee profile in the sidebar and both predictions update live.

## Deploy to Streamlit Community Cloud

1. Push this folder to a public (or private) GitHub repo, including the `models/` folder
   (or run `train.py` as a build step вЂ” see note below).
2. Go to [share.streamlit.io](https://share.streamlit.io), sign in with GitHub.
3. Click **New app**, pick your repo/branch, and set the main file path to `app.py`.
4. Click **Deploy**. Streamlit Cloud installs `requirements.txt` automatically.

**Note on model files:** `models/*.pkl` are binary artifacts. You can either:
- commit them directly to the repo (simplest вЂ” they're a few hundred KB), or
- add a one-line `python train.py` call at the top of `app.py` (or a Streamlit
  Cloud build hook) so the models are trained fresh on first deploy instead of
  being committed.

## How it works

- `train.py` loads `data/dataset.csv`, drops non-predictive columns
  (`EmployeeCount`, `EmployeeNumber`, `Over18`, `StandardHours` вЂ” these are
  constant or ID-like), builds a `ColumnTransformer` (StandardScaler for
  numeric features, OneHotEncoder for categorical features), wraps it in a
  `Pipeline` with the model, trains, evaluates on a held-out 20% split, and
  saves the fitted pipeline plus metadata (feature ranges/options, test
  metrics) needed to build the app's input form.
- `app.py` loads both pipelines with `@st.cache_resource`, dynamically builds
  a sidebar form from the saved metadata, and shows both predictions side by
  side whenever an input changes.

## Retraining on new data

Replace `data/dataset.csv` with a new file using the same column names/format
(tab-separated) and rerun `python train.py`. The app will pick up the new
models automatically on next load.
